# MVP 范围说明（InterviewMate RAG）

## 项目目标

构建一个可在本地快速演示的 **题库 + LLM 评卷** 技术面试练习小工具，用于简历展示：选题与难度、生成题目、提交答案后对照本题参考要点获得结构化评分与反馈。

## MVP 功能

### 已实现

- 外置 topic 白名单 + **多标签 `topics`**，与难度组合后从本地 JSON 种子中随机抽题（标签 OR）。
- **评卷不做向量检索**：仅用 `question_id` 对应的本题参考答案与要点构造 prompt；调用大模型按 rubric 输出 **JSON**（分数、亮点、缺失、改进回答）。
- 单页前端：选题、生成、展示要点与参考、填写答案、评估结果展示。

### 规划中（JD MVP，文档先行）

- 用户 **粘贴 JD 纯文本**；后端 **仅在此路径** 使用 **RAG（向量相似度）** 在指定难度下从题库 **组卷**（拟定接口 `POST /generate-paper-from-jd`）。
- 启动时在实现落地后将对全库条目批量 **Embedding**（用于组卷，**不用于评卷**）。
- **评卷仍为 LLM rubric + 本题 canonical**，不因 JD 组卷而引入检索邻居或 JD 全文进评卷 prompt。

## 非目标（刻意不做）

- 用户体系、鉴权、权限角色、管理后台。
- 支付、职位搜索、简历解析、大规模生产运维。
- 外部数据库（首版仅本地 JSON 文件）。

## 演示局限

- 需要可用的 **OpenAI API Key**（评卷 Chat；**JD 组卷实现后**另需启动阶段 Embedding 调用），无离线降级。
- 题库规模小，评分仅供演示，不代表真实面试评分标准。
- JD 组卷为 **相对排序**（Top‑N），不设额外语义阈值时仍返回池中相对最相近的题目。
- CORS 仅放行本地前端常用端口，非生产安全配置。
